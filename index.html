Old Business

The following items were carried over from previous discussions (prior to this team's formation):

    View discussion notes

    Ben and Evan debated the merits of React vs Vanilla JS for a small productivity app. The consensus leaned toward Vanilla JS to keep the bundle small, but a final vote will happen asynchronously in Slack.

  • Repository setup: GitHub org and repo need to be created by Nicole before end of week.
  • Team working agreement: draft was sent but not yet signed off by Bethany.

New Business

Core Features Discussed

The team identified the following must-have features for the MVP:

  1. Configurable work / break timer (default 25 / 5 minutes)
  2. Audio notification at the end of each interval
  3. Session counter (track number of completed pomodoros)
  4. Task list to associate with a session
  5. Dark mode toggle

Architecture Overview

Kaley sketched a high-level architecture diagram on the whiteboard.
The app will be a single-page application with three main components: Timer, Task Manager, and Settings Panel. See the Media section for the whiteboard photo.

Action Items

Decisions Made

The team agreed by unanimous vote that the app will be built with Vanilla JS, HTML, and CSS only — no frameworks for the initial MVP.
Next meeting is scheduled for Wednesday, April 15 at 5:00 PM via Zoom.

Miscellaneous Comments, Questions & Concerns

The following items were raised during open discussion:

  • Bethany asked whether we need accessibility compliance (WCAG AA).
    Response: Yes — Evan confirmed this is a grading requirement.
  • Kaley suggested adding an ambient sound feature (rain, white noise).
    Response: Moved to the "nice-to-have" backlog for post-MVP consideration.
  • Han flagged a potential issue with timer accuracy when the browser tab is in the background. Action item: Han to research Web Workers as a fix.

All concerns have been logged in the team's shared GitHub Meeting notes. Current status: pending review by team leads.
